Llenguatge de marques i sistemes de gestió de la informació

2008-10-02 00:00

Explicación

La práctica ha consistido en realizar una página html de un formulario común de registro. En primer lugar, he dividido el código html en dos artículos dentro de una misma sección. El primer artículo se ha destinado a la propia explicación y el segundo para el formulario en cuestión.

La explicación no tiene nada en particular, sólo un tag heading y distintos parráfos. El formulario lo he dividido en divs de clase form-group para crear bloques concretos para cada input. Al mismo tiempo, cada form-group constaba de un label y otro div de clase form-group-input. Esta estructura permite maquetar correctamente cada apartado del formulario. En lo referente a los fieldset, he utilizado dos para dividir la información solicitada. Creía que semánticamente sólo podía utilizarse un fieldset, pero si deseas dividir la información es una buena práctica incluir distintos fieldsets.

En lo referente a los estilos, he empezado con un código reset extraído en la web github que permite resetear todos los estilos propios de un navegador. De este modo, maquetaba con mis propias hojas el estilo (styles.css y form.css) el diseño exacto de mi página para cualquier navegador.

Por último, comentar que he dejado los colores para que el examinador pueda comprobar que todos los bloques han quedado bien definidos.
